# GitHub 偵察機 - 開發計畫

---

## 🔍 網頁運作的五個白話文步驟

### 第一步：你寫下名字（輸入框 + v-model 雙向繩子）

> 就像你在作業簿上寫「vue」這個名字

當你打字的時候，有一條「隱形繩子」把你的輸入框和一個「盒子」連在一起。你打什麼，盒子裡就裝什麼。

---

### 第二步：按下送出（按鈕 + v-if 開關）

> 就像你把作業交給老師

按下按鈕後，會觸發一個「打電話」的動作。同時，一個「開關」被打開，顯示「打電話中...」的提示。

---

### 第三步：快遞員出發（fetch 網路請求）

> 瀏覽器派了一個快遞員，帶著「幫我找 vue」的任務出門了

瀏覽器派快遞員去 GitHub 的公司，問：「有沒有人知道『vue』這個專案？」

---

### 第四步：抵達公司、找到東西（API 回應）

> 快遞員到了 GitHub 的倉庫大樓，員工在資料庫裡翻箱倒櫃

GitHub 在資料庫裡搜尋，找到了很多相關的專案，然後把結果（名字、描述、照片、星數）裝箱交給快遞員。

---

### 第五步：老師把作業發還給你（顯示在網頁上）

> 快遞員帶著資料跑回來，瀏覽器把結果顯示出來

瀏覽器收到資料後，把專案卡片一筆一筆顯示出來，你就看到 GitHub 偵察機幫你找到的結果了！

---

## ⏱️ 完整流程圖

```
你打字 → 按下按鈕 → 瀏覽器發請求 → GitHub收到 → GitHub找資料 → 
GitHub回傳 → 瀏覽器收到 → 網頁顯示結果
```

## 📚 重要指令的生活比喻

### v-model = 「雙向繩子」

想像：你有一個**遙控器**（輸入框）和一個**電視螢幕**（變數盒子）。

- 你按遙控器 → 電視跟著變
- 電視被別人改 → 遙控器也跟著變

**v-model** 就是那條連接兩邊的**隱形繩子**：

```html
<input v-model="searchName" />
```

---

### v-if = 「電源開關」

想像：你家有一個**電燈**（畫面上的元素）。

- 開關打開 → 燈亮 → 看得見
- 開關關掉 → 燈滅 → 看不見

**v-if** 就是那個**電源開關**：

```html
<p v-if="loading">打電話中...</p>
```

> 當 `loading` 是 `true`（正在打電話）→ 顯示「打電話中...」
> 當 `loading` 是 `false`（電話打完）→ 這行字消失

---

### v-for = 「影印機」

想像：你有一張**設計圖**，然後放進影印機。

- 按一次 → 印出一張卡片
- 按十次 → 印出十張卡片

**v-for** 就是那台**影印機**，可以根據資料數量，重複顯示同樣的元件：

```html
<div v-for="repo in repos" :key="repo.id">
  <!-- 這段會重複顯示很多次，取決於 repos 有多少筆資料 -->
</div>
```

## ✅ 已完成功能

| 功能 | 說明 |
|------|------|
| 輸入框 | 用 v-model 「記住」你打的字 |
| 搜尋按鈕 | 按下去會「打電話」給 GitHub |
| 載入中提示 | v-if 開關控制「打電話中...」顯示/隱藏 |
| 結果卡片 | v-for 影印機重複顯示每個專案 |
| 大頭照 | 顯示專案主人的圓形照片 |
| 深色主題 | 讓網頁看起來更像 GitHub |
| 查無此人 | 當搜尋不到結果時顯示提示 |
